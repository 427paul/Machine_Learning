#1. 인공지능의 분류 : ['지적 수준, '기능 발전', '구현 방식']

지적 수준 : 인간과 같은 사고의 기능 여부
약인공지능 : 특정 문제해결에 전문화된 인공지능
-Weak AI, Artificial Narrow Intelligence
-한 분야의 특정한 일을 인간의 지시에 따라 수행하는 인공지능
-어떠한 목적에 최적화된 알고리즘으로, 적당한 규칙에 의해 구현됨
예) 스팸메일 필터링, 검색 서비스, 구글번역, 유튜브 영상 추천, 알파고는 바둑, 테슬라의 자율주행차는 자율주행에만 최적화

강인공지능 : 모든 영역에서 인간과 같은 인공지능
-Strong AI, Artificial General Intelligence
-약인공지능의 제한된 기능을 뛰어넘어 모든 산업 분야에서 범용적으로 사용되는 인공지능
-지각력이 있고 스스로를 인식할 수 있는 존재
예) 영화 <<터미네이터>>에 등장하는 스카이넷, 비서로봇, 협동로봇(공장로봇)

초인공지능 : 인류 전체의 지능을 초월하는 인공지능
-Artificial Super Intelligence
-인간보다 몇 백배 이상 뛰어난 지능을 가진 존재
-연산 능력뿐만 아니라 과학 및 예술 분야에서 뛰어난 창의성을 발휘함
-일반적인 지식과 사회적인 능력도 인간을 뛰어넘는 지능을 가짐
예) '인류가 앞으로 1000년 동안 쓸 수 있는 에너지원 만들기'와 같은 고차원의 명령 수행 가능

특이점 : 인공지능의 발전이 가속화되어 모든 인류의 지성을 합친 것보다 더 뛰어난 초인공지능이 출현하는 시점

기능발전(레벨) : 입력에 따른 출력이 변하는 에이전트 관점
레벨1 : 단순 제어 프로그램
-단순한 제어 프로그램
-엄밀한 의미로 인공지능은 아님
-On/Off, Yes/No와 같이 필요한 변수가 두 가지 뿐인 가장 기초적인 인공지능
예) 세탁기, 전자면도기 등

레벨2 : 고전적 인공지능 : 입력과 출력의 조합 수가 극단적으로 많은 경우
-입력과 출력의 조합 수가 극단적으로 많은 경우
-추론 및 탐색을 하거나 기준의 지식베이스를 기반으로 판단
-단순 제어 프로그램과 크게 다르지 않은 알고리즘
-단순 제어 프로그램과의 차이는 패턴이 다양해진 것 뿐임
-경우의 수에 따라 행동하기 때문에 '학습'아 아닌 '정해진 규칙'에 따라 행동
예) 고전적 퍼즐, 로봇 청소기, 바둑 게임 프로그램

레벨3 : 머신러닝
-빅데이터를 바탕으로 자동 판단
-전형적으로 머신러닝 알고리즘 이용
-적절한 판단을 위해 '보유한 지식을 기반으로 추론 및 탐색하는 인공지능'
예) 지도학습, 비지도학습, 강화학습, 스마트폰의 얼굴인식, 유튜브의 맟춤 동영상 추천 시스템

레벨4 : 딥러닝이 도입된 인공지능
-입력데이터 자체를 학습해 스스로 판단하고 예측
-머신러닝보다 발전하여 데이터 자체를 스스로 학습하는 수준의 인공지능
-머신러닝을 이용한 학습과정에는 인간이 개입하지만(특징 추출),
-딥러닝을 이용한 학습 과정에서는 기계가 스스로 해답을 찾아 분류나 예측을 수행
예) 구글의 고양이 인식, 자율주행차와 지능형 CCTV

특징 추출(Feature Extraction) :
머신러닝에서 컴퓨터가 스스로 학습하려면 사람이 인지하는 데이터를 컴퓨터가 인지할 수 있는 데이터로 변환해야 함
-> 데이터별로 어떤 특징을 가지고 있는지를 찾아내고 그것을 토대로 데이터를 벡터로 변환하는 작업

구현방식 : 지적 기능 구현 방식
지식 기만 방법론(인지, 추론, 학습, 행동) : 저장된 지식을 기반으로 의사결정 수행 ['인지', '학습', '추론', '행동']
-인지, 학습, 추론, 행동과 같은 인간의 지적 기능을 모방하기 위해 '지적 기능들을 기호로 표현'하고 
이를 '논리적인 규칙에 근거하여 처리'함으로씨 문제를 해결하고자 하는 방법이 시도되었음

인지(Recognition) :
-보다 : 인간 수준 정확도 달성
-듣다 : 음성 인식/자연어 인식의 발전
-읽다 : 디지털화된 정보를 통해 물리적 환경인식
-컴퓨터가 특정 대상을 보고 듣고 읽을 수 있게 하려는 것
-인지의 종류 : ['자연어 처리', '통번역', '글씨 인식', '음석 인식', '이미지 인식', '동영상 인식']

학습(Learning) :
-인지한 환경 속에서 목적 달성을 위해 최적의 답을 찾아냄
-반복 학습을 통한 지식 고도화
-데이터를 이용하여 규칙(Rule)을 만들고 결론을 도출하는 영역
-머신러닝과 딥러닝이 여기에 속함
예) 강화형 머신러닝 알파고

추론(Mechanism) : 
-추론 인터페이스 : 학습된 지능에 기반하여 인지된 환경을 추론 및 예측
-데이터를 이용하여 규칙(Rule)을 만들고 결론을 도출하는 영역
-머신러닝과 딥러닝이 여기에 속함
예) 머신 이해력, 관계 네트워크

행동(Begavior) :
-행동 의사결졍 :
-문제를 스스로 발견하고 해결
-지능, 추론 능력에 기반한 자율적 판단과 행동 수행
-"기계가 인간과 같이 사고하고 행동하는 영역"
-매 순간 결정(Decision)을 내리고 행동할 수 있도록 전략(Policy)과 보상(Reward) 개념을 사용함
-딥마인드는 이러한 과정을 '소코반(Sokoban)' 게임을 이용하여 증명함
예) 완전 자율주행

데이터 기반 방법론(머신러닝 데이터마이닝) : 데이터로부터 추출된 지식으로 의사결정 수행 ['머신러닝', '데이터마이닝']
-제공된 데이터로부터 연역적으로 지식을 추출하여 문제를 해결하는 방법론

#2. 인공지능을 실현하기 위한 기술 요소

인공지능 서비스 개발 과정
1단계 : 데이터 수집
-사용자 문의
-다른 회사와의 협엽

2단계 : 데이터 저장 및 처리
-저장 및 처리 장치 이용
-클라우드 서비스 이용

3단계 : 프로그램 제작
-AI 알고리즘 활용
-패턴 분석 및 제작

4단계 : 활용 및 서비스 배포
-클라우드 서비스 활용

인공지능 서비스 개발을 위한 기술
-원활한 인공지능 구동을 위한 '하드웨어 기술'
-데이터 수집에 필요한 '사물인터넷 기술'
-데이터를 전송하기 위한 '5G 기술'
-데이터를 저장하고 처리하기 위한 '클라우드 기술'

사물 인터넷 센서 -> 5G를 이용한 연결 -> 데이터 전송 -> 데이터분석(클라우드->인공지능) ->서비스(디지털 트윈, 자율주행차, 스마트홈, 스마트시티)

#3. 인공지능을 위한 하드웨어 기술
GPU
-중앙처리장치(이하 CPU)만을 사용했던 시기에는 데이터를 처리하는 데 시간이 너무 많이 걸려서 실용적이지 못했음

1. GPU를 활용한 인공지능의 도약
-"GPU(Graphic Processing Unit)"
-그래픽 처리 장치
-컴퓨터의 그래픽 요소를 처리하기 위해 만들어진 기술
-실제 인공지능(딥러닝)에서의 학습은 훨씬 더 복잡한 연산들이 수행됨
-하지만 GPU 기술의 도입으로 딥러닝에서는 빠른 연산을 할 수 있게 되었고, 그 결과 현재의 인공지능까지 발전할 수 있었음

2. CPU와 GPU의 '성능 측면' 차이
CPU : 
-복잡한 계산을 코어의 개수만큼씩 처리하므로, 간단한 계산식을 여려 개 처리하려면 오래 걸림
-코어가 순차적으로 하나씩 계산함

GPU : 
-간단한 계산식을 동시에 빠르게 처리하지만, 하나의 복잡한 수식은 CPU보다 연산속도가 느림
-다수의 코어가 한번에 병렬로 처리함

		 CPU		GPU
코어(Core)수	적음		많음
작업 처리 방식	순차적		병렬적
연산 방식		소수의 연산	수천 개의 연산
장점		낮은 대기시간	높은 처리량

-처리해야 할 명령어와 데이터의 성격에 따라 때로는 CPU가, 때로는 GPU가 빠를 수 있음
-'인공지능의 학습'은 행렬곱,  미분과 같이 수식의 풀이가 중요하기 때문에 'GPU를 사용하는 것이 더 적합함'

#3. 인공지능을 위한 데이터 기술
사물인터넷(IoT : Internet of Things) : 각종 사물에 센서와 통신 기능을 내장하여 '인터넷에 연결하는 기술'
-무선통신을 통해 각종 사물과 연결됨

사물인터넷 기술의 3요소 
1.센싱(Sensing)기술 :
-사물이 가진 데이터를 인식하고 추출해 인터넷으로 전송하는 기술

2.유무선 네트워크 기술 :
-사람과 사물, 서비스 등의 분산된 컴퓨팅 자원을 유무선 통신으로 연결하여 고속처리, 병렬처리하는 기술

3.서비스 인터페이스 기술 :
-각종 서비스 분야와 형태에 알맞게 정보를 가공/처리/융합하는 기술

-5세대 이동통신
-LTE보다 수십 배 빠른 속도(20Gbps)와 안정성
-4차 산업혁명의 핵심 기술을 현실화시킬 통신 기술

초고속성 :
-5G의 속도는 LTE보다 20배 빠른 최대 20Gbps(LTE보다 100 많은 양의 데이터를 주고받음)

초연결성 :
-5G는 단위 면적(1km^2)당 접속 가능 기기 수가 100만 대를 넘어서기 때문에 사물인터넷 분야에서 5G 기술은 아주 활용도가 높음

초저저연성 :
-5G는 LTE 대비 10분의 1 수준인 1ms 지연시간을 가지므로, 끊김 없는 연결성을 제공함

#4. 인공지능을 위한 네트워크 기술
5G로 구현하는 인공지능
1. 초고속성 활용
-5G는 20Gbps의 속도를 제공함
-이렇게 빠른 속도를 활용하면 용량이 매우 큰 실감형 콘텐츠의 유통 및 서비스 활성화가 가능함
관련 예) 가상현실과 증상현실

2. 초연결성 활용
-다양한 스마트 기기의 연결 및 데이터 수집/제어/전송에느 5G의 초연결설 필수
-5G 시대에는 진정한 스마트홈 시대가 시작될 것임

3. 초저지연성 활용
-자율주행차 : 주변 도로 환경을 인지하고 즉각적으로 행동하기 위해 필요함
-스마트팩토리 : 물리적 시설을 가상으로 투영시킨 공장(디지털 트윈)을 통해 모든 현황을 실시간으로 모니터링 가능

#5. 인공지능을 위한 인프라 기술
클라우드 컴퓨팅(Cloud Computing)
-인터넷 어딘가에 존재하는 컴퓨팅 자원을 필요할 때마다 '컴퓨터나 스마트폰 등에 불러와 사용하는 서비스'
-구름을 뜻하는 클라우드(Cloud)라는 단어처럼 구름에 덮여 보이지 않는 듯한 컴퓨팅 자원을 원하는 대로 가져다 사용할 수 있음

클라우드의 장점
1.신속한 인프라 도입
-클라우드는 컴퓨팅 자원을 빌려 쓰겠다는 '신청서를 제출하면 10분 내로 인프라를 도입해서 서비스 개발 가능'
-인프라 도입 시간 절감
-서비스 제공 시기 단축 가능

2.유연한 인프라 관리
-기업에서 인프라를 직접 구매하여 구축할 경우에는 서비스를 이용할 사용자의 규모를 예측한 후 인프라를 도입해야 함
-수요를 예측하기 힘든 경우에는 클라우드 서비스를 사용하는 것이 편리함

클라우드의 단점
1.비싼 이용 비용
-클라우드는 '서비스 운영 비용이 높음'
-기업이 서비스를 제공함에 있어 비용을 가장 중요한 요소로 생각한다면, 클라우드는 적절한 대안이 아닐 수 있음

2.데이터 보관의 불안함
-보안상 클라우드 사업자가 고객의 데이터에 접근할 수 없을 뿐만 아니라 외부에 제공하지 않는다고 하지만, 기업 입장에서는 불안한 것이 사실임
-그래서 데이터의 외부 유출이 곤란한 기업이나 공공기관은 '클라우드 도입을 신중하게 결정'해야 함

인공지능의 한계 :
1.머신러닝의 99%는 인간의 수작업에 의존
-머신러닝을 유지하기 위해 많은 양의 데이터를 분류하고 정리하는 것이 필요함
-데이터를 분류하고 적절한 알고리즘을 제공하는 과정에는 사람이 개입해야 함
-결국 인공지능의 핵심은 인간의 노동력에 기반한다고 할 수 있음

				 [		    데이터 학습 과정		        ]
데이터 수집 과정 ->데이터 가공 과정 ->[데이터 학습 방법 선택 -> 매개변수 조정 -> 머신러닝 모델 학습] -> 머신러닝 모델 개발
-전 과정의 99%가 인간의 수작업으로 진행됨

2.편형된 데이터로 학습할 경우 인공지능(AI)도 편견 형성
-MS의 챗봇 '테이(Tay)'에 의해 네티즌들이 인종차별과 같은 모욕적인 단어들을 주입시킨 결과, 테이는 불쾌한 발언들을 쏟아내기 시작함
-이 사건을 통해 인간이 어떤 유형의 데이터를 제공하는지에 따라 인공지능도 편견을 형성할 수 있다는 결론이 나옴

기계가 편향되는 이유 :
-기술 프로그래밍에서 적절한 주의가 없으면 프로그래머의 편결도 결과에 미칠 가능성이 있음

관련 예) 악의적인 해커 침입으로 인한 기계 변형
